---
name: work-queue
description: Use when the user wants to break down a project, feature, or task into a structured queue of work items
---

- Produce a numbered list of work items that cover the complete scope of work
- Ensure the order of work items is logical and sequential
- Include a description with each work item
- Reproduce context from any linked supporting documents into the description of each work item. Everything needed to complete the work and nothing more
- Ensure each work item has all the information needed for a simple developer to complete it successfully
